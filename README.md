# Tempa – тепловизионный анализатор

Веб-приложение для построения карт максимальных температур из тепловизионных видеофайлов (формат `.ravi`).  
Обработка асинхронная, через Celery + Redis. Поддерживает интерактивную настройку порога и экспорт в PNG.

---

## 🚀 Быстрый старт (для локального ПК)

### Требования

- Python 3.12+ (рекомендуется 3.12)
- Git
- Redis (можно через WSL или установить отдельно)
- (Опционально) Docker

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/sdankov37/Tempa.git
cd Tempa
```

### 2. Создайте и активируйте виртуальное окружение

```bash

python -m venv venv
source venv/bin/activate        # Linux/macOS
venv\Scripts\activate           # Windows
```

### 3. Установите зависимости

```bash

pip install -r backend/requirements.txt
```

### 4. Настройте переменные окружения

Скопируйте .env.example в .env и отредактируйте при необходимости (секретный ключ, URL Redis и т.д.):

```bash

cp .env.example .env
```

### 5. Запустите Redis

## Вариант А – через WSL (Windows):

```bash

wsl sudo service redis-server start
```

## Вариант Б – установите Redis для Windows (скачайте с microsoftarchive/redis).

## Вариант В – через Docker:

```bash

docker run -d --name redis -p 6379:6379 redis:alpine
```

### 6. Запустите Celery worker (в отдельном терминале)

```bash

python -m celery -A backend.celery_app worker --loglevel=info --pool=solo
```

### 7. Запустите бэкенд (в другом терминале)

```bash

python -m uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000
```

### 8. Откройте интерфейс

# Перейдите в браузере по адресу:

👉 http://127.0.0.1:8000/
🐳 Запуск через Docker (для сервера)

Если у вас есть Docker и Docker Compose, запустите одной командой:

```bash
docker-compose up --build
```

### После запуска:

    Бэкенд: http://localhost:8000

    Фронтенд: http://localhost:8000 (раздаётся бэкендом)

### 🧪 Использование

    Зарегистрируйтесь – введите логин и пароль, нажмите «Зарегистрироваться».

    Войдите – после успешного входа вы попадёте на основную страницу.

    Загрузите файл – выберите .ravi (или .avi, .mp4), настройте параметры (T_min, T_max, порог).

    Нажмите «Запустить обработку» – статус задачи будет обновляться.

    Результат – после завершения отобразится тепловая карта.

    Изменяйте порог – двигайте слайдер, карта пересчитывается без повторной обработки.

    Сохраните результат – кнопка «Сохранить PNG» скачает карту в формате PNG с подписанной цветовой шкалой.

### 🛠 Технологии

    Бэкенд: FastAPI, SQLAlchemy, Celery, Redis, OpenCV, Matplotlib

    Фронтенд: HTML, CSS, Vanilla JS

    Контейнеризация: Docker, Docker Compose

    База данных: SQLite (легковесная, файл tempa.db)

### 📝 Примечания

    Формат .ravi – если файл не открывается как видео, processor.py пытается интерпретировать его как сырые 16-битные данные (экспериментально).

    Для больших файлов убедитесь, что у вас достаточно оперативной памяти и свободного места во временной папке.

    При первом запуске создаётся файл tempa.db (SQLite) – все данные пользователей и задач хранятся в нём.

### 🐛 Устранение неполадок

    401 Unauthorized – проверьте, что вы вошли в систему, и кука tempa_session сохранена (вкладка Application → Cookies в браузере).

    Ошибка 500 – посмотрите логи в терминале с Uvicorn и Celery. Чаще всего проблема в формате файла.

    Redis не запускается – проверьте, что Redis установлен и служба запущена. Альтернативно, замените REDIS_URL в .env на memory:// (но это не рекомендуется).

    Celery не видит задачи – убедитесь, что worker запущен с той же версией кода, что и бэкенд (перезапустите после обновлений).
